# Model Context Protocol (MCP) 核心内容提取（中译英对照）

## 一、MCP 核心定位与价值
1. 中文原文：在深入了解模型上下文协议（MCP）之前，理解构成其基础的关键概念和术语至关重要。本节将介绍该协议的基本理念，并为整个课程中讨论 MCP 实现提供通用术语。  
   English Translation: Before diving deeper into the Model Context Protocol, it’s important to understand the key concepts and terminology that form the foundation of MCP. This section will introduce the fundamental ideas that underpin the protocol and provide a common vocabulary for discussing MCP implementations throughout the course.

2. 中文原文：MCP 常被描述为“人工智能应用的 USB-C”。正如 USB-C 为各类外设与计算设备连接提供标准化的物理和逻辑接口一样，MCP 为人工智能模型与外部功能的连接提供了统一协议。  
   English Translation: MCP is often described as the “USB-C for AI applications.” Just as USB-C provides a standardized physical and logical interface for connecting various peripherals to computing devices, MCP offers a consistent protocol for linking AI models to external capabilities.

3. 中文原文：这种标准化使整个生态系统受益：用户在不同人工智能应用中获得更简单、更一致的体验；人工智能应用开发者可轻松集成不断扩展的工具和数据源生态；工具和数据提供商只需创建一个可适配多个人工智能应用的实现；更广泛的生态系统则因互操作性提升、创新加速和碎片化减少而受益。  
   English Translation: This standardization benefits the entire ecosystem: users enjoy simpler and more consistent experiences across AI applications; AI application developers gain easy integration with a growing ecosystem of tools and data sources; tool and data providers need only create a single implementation that works with multiple AI applications; the broader ecosystem benefits from increased interoperability, innovation, and reduced fragmentation.


## 二、MCP 解决的集成问题
1. 中文原文：“M×N 集成问题”指的是在缺乏标准化方案的情况下，将 M 个不同的人工智能应用与 N 个不同的外部工具或数据源连接时面临的挑战。  
   English Translation: The “M×N Integration Problem” refers to the challenge of connecting M different AI applications to N different external tools or data sources without a standardized approach.

2. 中文原文：若没有 MCP 这类协议，开发者需要创建 M×N 个自定义集成——即人工智能应用与外部功能的每一种可能组合都需单独开发。这一过程复杂且成本高昂，给开发者带来大量阻力，同时导致维护成本居高不下。  

   English Translation: Without a protocol like MCP, developers would need to create M×N custom integrations—one for each possible pairing of an AI application with an external capability. This is a very complex and expensive process which introduces a lot of friction for developers, and high maintenance costs.


<img width="682" height="324" alt="image" src="https://github.com/user-attachments/assets/96adeabe-6c7d-4058-9800-e89e4a34725a" />


3. 中文原文：当存在多个模型和多个工具时，集成数量会变得难以管理，且每个集成都有独特的接口。  

   English Translation: Once we have multiple models and multiple tools, the number of integrations becomes too large to manage, each with its own unique interface.

4. 中文原文：MCP 通过提供标准接口，将该问题转化为 M+N 问题：每个人工智能应用只需实现一次 MCP 客户端，每个工具或数据源只需实现一次 MCP 服务端。这极大降低了集成复杂度和维护负担。  

   English Translation: MCP transforms this into an M+N problem by providing a standard interface: each AI application implements the client side of MCP once, and each tool/data source implements the server side once. This dramatically reduces integration complexity and maintenance burden.

## 三、MCP 核心术语

<img width="698" height="138" alt="image" src="https://github.com/user-attachments/assets/a50e2640-1b14-4591-86f4-c2d32cbd6f07" />


1. 中文原文：MCP 与 HTTP 的客户端 - 服务器关系类似，也包含客户端和服务器两个角色。  
 
   English Translation: Just like client-server relationships in HTTP, MCP has a client and a server.

2. 中文原文：**主机（Host）**：用户直接交互的面向用户的人工智能应用。例如 Anthropic 的 Claude Desktop、Cursor 等人工智能增强型集成开发环境（IDE）、Hugging Face Python SDK 等推理库，或基于 LangChain、smolagents 等库构建的自定义应用。主机负责发起与 MCP 服务器的连接，并协调用户请求、大语言模型（LLM）处理和外部工具之间的整体流程。  

   English Translation: **Host**: The user-facing AI application that end-users interact with directly. Examples include Anthropic’s Claude Desktop, AI-enhanced IDEs like Cursor, inference libraries like Hugging Face Python SDK, or custom applications built in libraries like LangChain or smolagents. Hosts initiate connections to MCP Servers and orchestrate the overall flow between user requests, LLM processing, and external tools.

3. 中文原文：**客户端（Client）**：主机应用中负责管理与特定 MCP 服务器通信的组件。每个客户端与单个服务器维持 1:1 连接，处理 MCP 通信的协议层细节，并作为主机逻辑与外部服务器之间的中介。  

   English Translation: **Client**: A component within the host application that manages communication with a specific MCP Server. Each Client maintains a 1:1 connection with a single Server, handling the protocol-level details of MCP communication and acting as an intermediary between the Host’s logic and the external Server.

5. 中文原文：**服务器（Server）**：通过 MCP 协议对外提供功能（工具、资源、提示词）的外部程序或服务。  

   English Translation: **Server**: An external program or service that exposes capabilities (Tools, Resources, Prompts) via the MCP protocol.

7. 中文原文：很多资料会将“客户端（Client）”和“主机（Host）”混用。从技术角度而言，主机是面向用户的应用，客户端则是主机应用中负责管理与特定 MCP 服务器通信的组件。  
   English Translation: A lot of content uses “Client” and “Host” interchangeably. Technically speaking, the host is the user-facing application, and the client is the component within the host application that manages communication with a specific MCP Server.

8. 中文原文：MCP 是一种类似 HTTP 或 USB-C 的标准，也是用于连接人工智能应用与外部工具及数据源的协议。因此，使用标准术语对 MCP 的有效运行至关重要。在为应用编写文档和与社区沟通时，应使用以下术语。  
   English Translation: MCP is a standard like HTTP or USB-C, and is a protocol for connecting AI applications to external tools and data sources. Therefore, using standard terminology is crucial to making the MCP work effectively. When documenting our applications and communicating with the community, we should use the following terminology.

## 四、MCP 核心功能（Capabilities）
1. 中文原文：当然，应用的价值在于其提供的功能总和。因此，功能是应用中最重要的部分。MCP 可连接任何软件服务，但有一些常见功能适用于众多人工智能应用。  
   English Translation: Of course, your application’s value is the sum of the capabilities it offers. So the capabilities are the most important part of your application. MCP can connect with any software service, but there are some common capabilities that are used for many AI applications.

2. 中文原文：**工具（Tools）**：人工智能模型可调用以执行操作或获取计算数据的可执行函数，通常与应用的用例相关。例如，天气应用的工具可能是一个返回特定地点天气的函数。  
   English Translation: **Tools**: Executable functions that the AI model can invoke to perform actions or retrieve computed data. Typically relating to the use case of the application. Example: A tool for a weather application might be a function that returns the weather in a specific location.

3. 中文原文：**资源（Resources）**：无需大量计算即可提供上下文的只读数据源。例如，科研助手可能包含一个用于存储科学论文的资源。  
   English Translation: **Resources**: Read-only data sources that provide context without significant computation. Example: A researcher assistant might have a resource for scientific papers.

4. 中文原文：**提示词（Prompts）**：指导用户、人工智能模型与可用功能之间交互的预定义模板或流程。例如，一个摘要生成提示词。  
   English Translation: **Prompts**: Pre-defined templates or workflows that guide interactions between users, AI models, and the available capabilities. Example: A summarization prompt.

5. 中文原文：**采样（Sampling）**：由服务器发起的、请求客户端/主机执行大语言模型交互的操作，支持递归行为——即大语言模型可审查生成的内容并做出进一步决策。例如，写作应用审查自身输出后决定进一步优化内容。  
   English Translation: **Sampling**: Server-initiated requests for the Client/Host to perform LLM interactions, enabling recursive actions where the LLM can review generated content and make further decisions. Example: A writing application reviewing its own output and decides to refine it further.

   English Translation: In the following diagram, we can see the collective capabilities applied to a use case for a code agent.

7. 中文原文：该应用可能通过以下方式使用其 MCP 实体：  
   - 工具（Tool）：代码解释器（Code Interpreter）——可执行大语言模型编写的代码的工具。  
   - 资源（Resource）：文档（Documentation）——包含应用文档的资源。  
   - 提示词（Prompt）：代码风格（Code Style）——指导大语言模型生成代码的提示词。  
   - 采样（Sampling）：代码审查（Code Review）——允许大语言模型审查代码并做出进一步决策的采样功能。  
   English Translation: This application might use their MCP entities in the following way:
   - Tool: Code Interpreter – A tool that can execute code that the LLM writes.
   - Resource: Documentation – A resource that contains the documentation of the application.
   - Prompt: Code Style – A prompt that guides the LLM to generate code.
   - Sampling: Code Review – A sampling that allows the LLM to review the code and make further decisions.
